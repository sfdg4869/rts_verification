DB IDINSTANCEHOST IPHOST NAME DB USERSIDLSNR PORTOS

3 SSH 접속 정보 & 옵션

※ SSH 정보는 Target 점검 실행(RTS 7단계)에서만 사용됩니다. Repo 점검 실행에는 사용되지 않습니다.

4 Target DB 접속정보 (쿼리 실행 전용)

※ Step 2에서 선택한 Target DB 정보(host/port/user/sid)를 사용합니다. 비밀번호만 입력하세요.
NO TARGET Step 2에서 Target DB를 선택해주세요

5 점검 실행

Target 점검 항목
1. 로그 디렉터리 2. rtsctl stat 3. PID 교차검증 4. ERROR grep 5. CPU/MEM 6. 비정상 종료 시그널 7. Target 조회(v$sql)
Repo 점검 항목
1. APM_TOP_OS_PROCESS_LIST 누락 재현 여부 2. ORA_SQL_ELAPSE 1초 이상 로깅 3. ORA_SQL_STAT_10MIN 50초 이상 로깅 4. ORA_SQL_ELAPSE Elapsed/Execution 로깅 5. ORA_SQL_ELAPSE 대상 SQL 수집 확인 6. ORA_SQL_STAT_10MIN 대상 SQL 수집 확인
Repo 점검 실행은 Step 1 Repo DB 연결값만 사용합니다.

CPU / MEM 전용 조회

Step 2 Target + Step 3 SSH 정보를 사용해 현재값/2시간 집계를 조회합니다. (in-memory 저장)
