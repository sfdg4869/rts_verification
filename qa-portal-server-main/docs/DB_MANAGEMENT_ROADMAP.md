```
    │
    └─→ Target DB 선택

            ├─→ Repo DB의 APM_DB_INFO에서 db_id로 조회
            │
            └─→ shared_db에 Target DB 설정 저장
```

## 🔑 주요 특징

### ✅ Repo DB 관리
- **저장 위치**: MongoDB `repo_test` 데이터베이스의 `qs1_db` 컬렉션
- **구조**: `entries` 배열에 모든 필드 포함
- **CRUD**: 생성(POST), 수정(PUT), 삭제(DELETE), 조회(GET)

### ✅ 프로파일 관리
- **저장 위치**: MongoDB `connection_profiles` 컬렉션
- **참조**: `repo_config_id`로 MongoDB entries 참조
- **특징**: Target DB 정보는 포함하지 않음 (APM_DB_INFO에서 선택)

### ✅ Target DB 관리
- **저장 위치**: 메모리 (`shared_db._db_connections['target']`)
- **소스**: Repo DB의 `APM_DB_INFO` 테이블
- **선택 방식**: `db_id`로 선택

### ✅ 설정 로드
- **시작 시**: MongoDB에서 기본 프로파일 로드
- **프로파일 변경 시**: `set_connection_profile()` 호출
- **Target DB 선택 시**: `POST /rts/mongodb/target-db/select` 호출

## 🎯 사용 시나리오

### 시나리오 1: 새로운 Repo DB 추가
```
1. POST /rts/mongodb/configs
   → Repo DB 정보 MongoDB에 저장
   → config_id 반환

2. POST /rts/mongodb/profiles
   → 프로파일 생성 (위 config_id 사용)
```

### 시나리오 2: Target DB 사용 준비
```
1. POST /rts/mongodb/profiles/<name>/activate
   → 프로파일 활성화
   → Repo DB 설정 로드

2. POST /rts/mongodb/target-db/list
   → Repo DB의 APM_DB_INFO에서 Target DB 목록 조회

3. POST /rts/mongodb/target-db/select
   → db_id로 Target DB 선택
   → shared_db에 저장
```

### 시나리오 3: 데이터 조회
```
1. 애플리케이션 시작
   → MongoDB에서 기본 프로파일 로드
   → Repo DB 설정 메모리에 저장

2. API 호출 시
   → shared_db.get_connection('repo')
   → shared_db.get_connection('target')
   → 각각의 DB에 연결하여 데이터 조회
```
